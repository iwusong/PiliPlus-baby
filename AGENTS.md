# AGENTS.md

## First Principles

- **API 驱动**：整个应用是 BiliBili API 的壳。`http/` 和 `grpc/` 是地基，任何改动先想清楚对 API 层的影响。
- **Fork 优先**：维护者刻意 fork 多个依赖（GetX、media_kit、window_manager 等）而非用上游，说明需要在框架层控制行为。不要试图绕过 fork，尊重其存在。
- **存储即状态**：Hive boxes 承载用户数据、设置、登录态。`GStorage.init()` 失败会直接 `exit(0)`，修改存储相关代码必须考虑向后兼容和数据迁移。
- **视频播放是核心**：所有功能围绕视频播放展开。`plugin/pl_player/` 是自定义播放器，`media_kit` 是底层引擎，两者都经过深度定制。
- **页面自包含**：115+ 页面目录各自独立（view/controller/logic），新增功能遵循同样模式，不要跨页面耦合。
- **Flutter SDK 可被修补**：`lib/scripts/*.patch` 直接修改 Flutter 源码来绕过框架 bug，这是有意为之——不在应用层做 workaround。
- **渐进式重构**：`models/` → `models_new/` 是正在进行的迁移。新代码用 `models_new/`，不要回头改 `models/` 的结构。
- **平台差异是常态**：`PlatformUtils.isMobile` / `isDesktop` 贯穿全局，每个功能都要考虑多端表现。

## Project Overview

PiliPlus is a BiliBili third-party client built with Flutter. Targets Android, iOS, Windows, Linux, and macOS.

## Toolchain

- **Flutter**: 3.44.0 (pinned in `.fvmrc`; use `fvm flutter` if FVM is installed, otherwise ensure global Flutter matches)
- **Dart SDK**: >=3.12.0
- **State management / routing**: GetX (`get` package, forked)
- **Local storage**: Hive CE (`hive_ce`)
- **HTTP**: Dio + HTTP/2 adapter
- **Video playback**: media_kit (forked, with per-platform overrides in `pubspec.yaml`)
- **gRPC/Protobuf**: `protobuf` package; generated `.pb.dart` files live in `lib/grpc/bilibili/`

## Build Config & Versioning

- `lib/build_config.dart` reads build-time defines (`pili.code`, `pili.name`, `pili.hash`, `pili.time`) via `--dart-define-from-file`.
- `pili_release.json` is generated by `lib/scripts/build.ps1` (CI only) and is gitignored. For local dev builds without it, defaults apply (`SNAPSHOT`, code `1`).
- `lib/scripts/patch.ps1` applies Flutter framework patches before CI builds. These patch `$FLUTTER_ROOT` source files — **do not run locally** unless you understand the side effects. Patches fix known Flutter issues (bottom sheet, scroll view, text selection, navigator, etc.).

## Architecture

```
lib/
  main.dart              # App entry point
  build_config.dart      # Build-time version/commit defines
  router/app_pages.dart  # All GetX route definitions (~200 routes)
  pages/                 # Feature pages (~115 dirs), each with view/controller/logic
  http/                  # API layer (Dio-based), one file per domain (video, search, live, etc.)
  grpc/                  # gRPC clients; bilibili/ contains generated protobuf Dart code
  models/                # Hand-written data models (some with .g.dart json_serializable)
  models_new/            # Newer model layer (~42 domains)
  services/              # App services (account, download, audio, logging)
  utils/                 # Utilities (storage, signing, theme, platform detection)
  common/                # Shared widgets, constants, styles
  plugin/pl_player/      # Custom player plugin
  tcp/live.dart          # Live stream TCP/WebSocket handling
  scripts/               # CI build & patch PowerShell scripts
```

- Entry point: `lib/main.dart`
- Routes: `lib/router/app_pages.dart` — all pages registered via GetX `GetPage`
- API calls: `lib/http/` — each file wraps a BiliBili API domain
- Storage: `GStorage` class in `lib/utils/storage.dart` initializes all Hive boxes

## Important Conventions

- **Import style**: Always use `package:PiliPlus/...` absolute imports (enforced by `always_use_package_imports` lint). Never use relative `../` imports across directories.
- **Generated code**: `lib/grpc/bilibili/**` is excluded from analysis in `analysis_options.yaml`. Do not edit those files manually.
- **Lint rules**: Project uses `flutter_lints` plus many additional strict rules (see `analysis_options.yaml`). Key ones: `prefer_const_constructors`, `avoid_print`, `always_declare_return_types`, `cascade_invocations`.
- **Trailing commas**: `formatter: trailing_commas: preserve` — do not force-add or remove trailing commas; respect existing style.
- **Forked dependencies**: Many packages point to the maintainer's GitHub forks (GetX, media_kit, window_manager, etc.). When updating deps, check `pubspec.yaml` git refs carefully.

## Platforms

- **Android**: Requires Java 17, signing config via `android/key.properties` (gitignored)
- **iOS**: Standard Flutter iOS build
- **Windows / Linux / macOS**: Desktop builds with window management (`window_manager`)

## Gotchas

- `GStorage.init()` failure calls `exit(0)` — if Hive boxes corrupt, the app silently exits.
- The app applies patches to the Flutter SDK source in CI. If you see unexpected framework behavior, check `lib/scripts/*.patch`.
- `lib/utils/wbi_sign.dart` implements BiliBili's WBI signature algorithm — changes here can break all API requests.
- `models/` and `models_new/` coexist. New model work goes in `models_new/`.
- `.gitignore` excludes `test*` — tests are not tracked in this repo.
